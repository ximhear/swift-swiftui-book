# Swift/SwiftUI 중고급 서적 프로젝트

## 프로젝트 개요
Swift 및 SwiftUI 중고급 개발자를 대상으로 한 기술 서적을 집필하는 프로젝트입니다.

## 언어
- 본문은 **한국어**로 작성합니다.
- 코드 주석은 한국어로 작성하되, 변수/함수명은 영문을 사용합니다.
- 기술 용어는 널리 통용되는 영문 원어를 병기합니다. (예: "제네릭(Generic)")

## 대상 독자
- Swift 기본 문법을 이해하고 있는 중급 이상 개발자
- SwiftUI로 간단한 앱을 만들어본 경험이 있는 개발자
- 더 깊은 이해와 실무 패턴을 원하는 개발자

## 집필 원칙

### 콘텐츠 품질
- **왜(Why)**를 먼저 설명하고, **어떻게(How)**를 이어서 보여준다.
- 공식 문서에 없는 실무 인사이트와 함정(pitfall)을 적극적으로 다룬다.
- 각 장은 독립적으로 읽을 수 있되, 전체 흐름이 자연스럽게 이어지도록 구성한다.
- 추상적 설명보다 구체적인 코드 예제를 우선한다.

### 코드 예제
- 모든 코드는 **Swift 6.1 / Xcode 16** 기준으로 작성한다.
- 최소한의 완전한 예제(Minimal Complete Example)를 원칙으로 한다.
- 코드에는 핵심 포인트를 설명하는 주석을 포함한다.
- 잘못된 코드 → 올바른 코드 비교 패턴을 적극 활용한다.
- Preview를 활용한 시각적 확인이 가능한 예제를 선호한다.

### 구조 및 포맷
- 각 장(Chapter)은 `chapters/` 디렉토리 아래 개별 디렉토리로 관리한다.
- 각 장 디렉토리에는 본문 마크다운과 예제 코드 파일을 함께 둔다.
- 장 번호는 `ch01`, `ch02` 형식을 사용한다.
- 그림/다이어그램 설명이 필요한 곳에는 `[그림: 설명]` 플레이스홀더를 사용한다.

## 디렉토리 구조

```
swift-swiftui-book/

│   ├── ch01-swift-type-system/
│   │   ├── chapter.md         # 본문
│   │   └── examples/          # 예제 코드(.swift)
│   ├── ch02-.../
│   └── ...

├── assets/                    # 공용 이미지/다이어그램

```

## 에이전트 작업 지침

### 새 장 작성 시
1. `outline.md`에서 해당 장의 위치와 전후 맥락을 확인한다.
2. 장 디렉토리와 `chapter.md`, `examples/` 디렉토리를 생성한다.
3. 도입부에서 이 장을 왜 배워야 하는지 동기를 부여한다.
4. 핵심 개념 → 실습 예제 → 심화 내용 → 정리 순으로 전개한다.
5. 예제 코드는 `examples/` 디렉토리에 별도 `.swift` 파일로도 저장한다.

### 기존 장 수정 시
1. 반드시 기존 내용을 먼저 읽고 전체 흐름을 파악한다.
2. 수정 범위를 최소화하고, 기존 설명 스타일과 톤을 유지한다.
3. 새로운 코드 예제 추가 시 기존 예제와의 일관성을 확인한다.

### 코드 리뷰 요청 시
- Swift 6.1 문법 및 컴파일 가능 여부를 확인한다.
- Strict Concurrency 체크를 기본으로 적용한다.
- SwiftUI의 최신 API 사용 여부를 확인한다.
- 더 나은 패턴이 있으면 제안한다.
